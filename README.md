1. Esse projeto foi desenvolvido a partir do desafio sobre enchentes no Brasil. A minha ideia surgiu ao perceber a dificuldade das pessoas em encontrar abrigos disponíveis durante situações de emergência, com esse projeto simples consegui facilitar o cadastro de cada abrigo.

2. O problema escolhido foi a falta de informação sobre abrigos, onde muitas pessoas não sabem para onde ir ou acabam indo para locais já lotados.

3. A solução consiste em um sistema que organiza informações sobre abrigos, permitindo visualizar capacidade e cadastrar novos locais.

4. Estrutura do Sistema:

# Front-end
Interface simples em HTML, CSS e JavaScript para listar e cadastrar abrigos.

# Back-end
API desenvolvida com Node.js e Express.

# Banco de Dados
PostgreSQL para armazenar informações dos abrigos.
